# 导出格式详解

Narrafiilm 支持多种导出格式，覆盖从快速分享到专业后期的全场景需求。

---

## 快速导出

适合直接分享到社交平台，快速出片。

### 格式参数

| 参数 | 可选值 | 默认值 |
|------|--------|--------|
| 格式 | MP4 (H.264)、MP4 (H.265)、WebM | MP4 (H.264) |
| 分辨率 | 4K / 1080p / 720p / 480p / 原始 | 1080p |
| 帧率 | 24 / 25 / 30 / 60 fps | 30 fps |
| 码率 | 自动 / 5 Mbps / 10 Mbps / 20 Mbps / 自定义 | 自动 |
| 音频 | AAC 128k / AAC 192k / AAC 320k / 无音频 | AAC 192k |

### 预设

| 预设名称 | 分辨率 | 帧率 | 码率 | 适用场景 |
|----------|--------|------|------|----------|
| **抖音/快手** | 1080×1920 (9:16) | 30 fps | 10 Mbps | 短视频平台 |
| **小红书** | 1080×1350 (4:5) | 30 fps | 8 Mbps | 图文类平台 |
| **B站/YouTube** | 1920×1080 (16:9) | 60 fps | 15 Mbps | 视频平台 |
| **微信视频号** | 1080×1920 (9:16) | 30 fps | 8 Mbps | 社交平台 |
| **微博** | 1920×1080 (16:9) | 30 fps | 10 Mbps | 社交平台 |

## 专业格式导出

适合导入到专业剪辑软件进行二次编辑。

### 剪映 Draft JSON

导出剪映草稿格式，可在剪映中直接打开继续编辑。

::: tip 💡 推荐场景
用 Narrafiilm 做 AI 创作 + 剪映做精细调校是目前最高效的工作流。
:::

```yaml
导出选项:
  格式: Draft JSON
  包含素材: ✅ (打包所有引用的视频/音频文件)
  字幕格式: SRT (内嵌)
  音轨: 独立音轨
```

### Adobe Premiere XML

导出 Premiere Pro 交换格式（`.fcpxml` → Premiere）。

```yaml
导出选项:
  格式: Adobe Premiere XML
  版本: Premiere Pro 2023+
  时间码: 25fps / 29.97fps / 30fps
  包含转场: ✅
  包含特效: ✅
```

### Final Cut Pro FCPXML

导出 Final Cut Pro 格式。

```yaml
导出选项:
  格式: FCPXML
  版本: Final Cut Pro 10.6+
  包含音频: ✅
  颜色空间: Rec.709
```

### DaVinci Resolve XML

导出 DaVinci Resolve 格式。

```yaml
导出选项:
  格式: DaVinci Resolve XML
  版本: DaVinci Resolve 18+
  包含静音轨: ✅
  颜色管理: 跟随项目设置
```

## 纯视频/音频导出

### 仅导出视频

去掉 AI 添加的配音和字幕，导出纯净的 AI 处理后视频：

```yaml
导出选项:
  类型: 仅视频流
  包含字幕: ❌
  包含配音: ❌
```

### 仅导出音频

单独导出 AI 生成的配音：

| 格式 | 质量 | 适用场景 |
|------|------|----------|
| MP3 | 128k / 192k / 320k | 通用 |
| WAV | 无损 | 音频编辑 |
| AAC | 256k | 高质量存档 |

### 仅导出字幕

单独导出字幕文件：

| 格式 | 说明 |
|------|------|
| `.srt` | 通用字幕格式，兼容性最好 |
| `.ass` | 支持高级样式（描边、阴影、动画） |
| `.vtt` | Web 字幕格式 |
| `.lrc` | 歌词格式（用于音乐视频） |

---

## HDR 导出

适合在 HDR 设备上播放的专业导出选项：

```yaml
导出选项:
  色彩空间: Rec.2020 (HDR)
  HDR 标准: HDR10 / HLG / Dolby Vision
  峰值亮度: 1000 nits / 4000 nits
  色深: 10-bit / 12-bit
```

::: warning ⚠️ HDR 限制
- 输入素材必须是 HDR 格式
- Dolby Vision 导出需要支持的硬件
- 部分平台（抖音、微信）不支持 HDR 上传
:::

## 高级参数

### 编码器选项

#### H.264 (推荐)

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `profile` | 编码档次 | High |
| `level` | 编码级别 | 4.1 |
| `preset` | 编码速度/质量权衡 | medium |
| `crf` | 质量控制 (0-51，越低质量越好) | 18-23 |
| `tune` | 调优选项 | film / animation / grain |

#### H.265 (HEVC)

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `profile` | 编码档次 | Main 10 |
| `preset` | 编码速度/质量权衡 | medium |
| `crf` | 质量控制 | 20-25 |

### 音频编码选项

| 参数 | 说明 | 可选值 |
|------|------|--------|
| 编码器 | 音频编码器 | AAC / MP3 / OPUS / PCM |
| 采样率 | 音频采样率 | 44.1kHz / 48kHz |
| 声道 | 声道配置 | 单声道 / 立体声 / 5.1 环绕 |

## 文件命名与输出

### 自动命名

```
{project}_{mode}_{date}_{resolution}.{ext}
# 示例: travel_vlog_ai_commentary_20260331_1080p.mp4
```

### 自定义命名

支持模板变量：

| 变量 | 说明 | 示例 |
|------|------|------|
| `{name}` | 项目名称 | travel_vlog |
| `{mode}` | 创作模式 | ai_commentary |
| `{date}` | 导出日期 | 2026-03-31 |
| `{time}` | 导出时间 | 14_30_00 |
| `{resolution}` | 分辨率 | 1080p |
| `{fps}` | 帧率 | 30fps |
| `{index}` | 序号（批量导出时） | 001 |

### 批量导出

一次导出多种格式/分辨率：

```yaml
批量导出预设:
  - 抖音版: 1080×1920, 10 Mbps
  - B站版: 1920×1080, 60 fps, 15 Mbps
  - 存档版: 4K, 无压缩, 无音频
```

## 导出性能

| 视频时长 | 分辨率 | 推荐编码器 | 预计耗时 |
|----------|--------|------------|----------|
| < 5 分钟 | 1080p | H.264 (medium) | 1-3 分钟 |
| 5-30 分钟 | 1080p | H.264 (medium) | 5-15 分钟 |
| > 30 分钟 | 4K | H.265 (slow) | 30-60 分钟 |

### 加速导出

- ✅ 使用 NVIDIA GPU（可加速 3-5 倍）
- ✅ 使用 SSD 作为输出目录
- ✅ 关闭其他程序
- ✅ 使用 H.264 代替 H.265（速度更快）

## 常见问题

### ❓ 导出的文件太大？

1. 使用 H.265 编码（体积减少约 40%）
2. 降低码率或使用 CRF 模式
3. 降低分辨率

### ❓ 导入剪映报错？

确保导出了完整的 Draft JSON（包含素材打包），剪映 10.0+ 版本兼容性最好。

### ❓ Premiere 导入颜色不对？

导入后在 Premiere 中应用 Rec.709 色彩空间转换。

::: tip 💡 最佳实践
1. AI 创作用 Narrafiilm，精细调校用剪映/Pr
2. 重要项目导出时选择"包含素材包"防止文件丢失
3. 最终发布前用原始分辨率导出一次存档
:::

::: info 💡 Pro Tip
如果你使用 DaVinci Resolve，建议导出 XML 后在 Resolve 中重新链接媒体，而不是直接导出视频，这样可以保留最大的后期调色空间。
:::
